в—Ҹ The information-theoretic frame

  Two documents are related exactly when they are not independent: knowing
  something about one reduces your uncertainty about the other. Relatedness is
  mutual information between documents along some channel, and the unit of
  relatedness is a shared (or linked) feature.

  The single most important consequence вҖ” and the thing that should govern every
  score in Hoplite вҖ” is that signal strength is the self-information of the 
  shared feature, not the fact of sharing. Two docs sharing the term "the" carry
  ~0 bits; two docs sharing "recursive-CTE traversal" carry many. This is
  pointwise mutual information: a co-occurrence is surprising (informative) in
  proportion to how improbable the shared feature is on its own. Every latent
  signal below is really "co-occurrence of a high-entropy feature," and the
  threshold knob your doc mentions is, formally, a cut on PMI.

  So the full enumeration is: every feature space over which documents are 
  statistically non-independent is a relatedness channel. Here they are.

  The enumeration

  I'll group by the feature space that carries the signal, and tag each:
  Explicit (authored) vs Latent (emergent); directed vs undirected; binary vs
  graded.

  1. Declared linkage вҖ” the link graph's values

  The author asserted the relationship; the signal is simply true.
  - Direct reference вҖ” A points to B. (E, directed, binary)
  - Backlink вҖ” the same edge read in reverse; who points to B. (E, directed, 
  binary) вҖ” one assertion, two readings.
  - Typed/stereotyped edge вҖ” the link carries a label (supersedes, derives-from,
  contradicts, refutes). The kind is itself a feature. (E, directed, binary)
  - Ghost reference вҖ” a link whose target doesn't exist yet; relates the citing
  doc to an intended one. (E, directed, binary)
  - Transclusion/embedding вҖ” one doc includes another's content; strictly
  stronger than reference. (E, directed, binary)

  2. Declared classification вҖ” metadata values

  - Shared tag вҖ” classification kinship; relate by kind even when topics
  diverge. (E, undirected, graded by tag rarity)
  - Shared structured property вҖ” same project, status, author frontmatter
  field/value. (E, undirected, graded by value rarity)
  - Title/identifier overlap вҖ” shared distinctive name tokens. (E/L, undirected,
  graded)

  3. Content вҖ” the words themselves

  - Lexical overlap (FTS/TF-IDF) вҖ” shared rare terms, exact n-gram coincidence.
  (L, undirected, graded) вҖ” this is PMI made literal: IDF is self-information.
  - Semantic/embedding similarity вҖ” closeness in distributional-meaning space;
  catches paraphrase the words miss. (L, undirected, graded by distance)
  - Topic-model kinship вҖ” share a latent topic distribution (LDA-style). (L, 
  undirected, graded)
  - Named-entity co-mention вҖ” reference the same person, system, file,
  identifier, ticket. (L, undirected, graded by entity rarity)
  - Near-duplication / quotation вҖ” verbatim or near-verbatim passages; the
  redundancy your "corpus repeats itself" problem describes. (L, undirected, 
  graded)
  - Shared external citation вҖ” both cite the same outside URL/source
  (bibliographic coupling on external refs). (L, undirected, graded)

  4. Graph topology вҖ” the link graph's structure (read off the edges, not the 
  values)

  This is the family your doc calls structural kinship; it has more members than
  co-citation.
  - Co-citation вҖ” both pointed to by the same third doc. (L, undirected, graded 
  by count)
  - Bibliographic coupling вҖ” both point to the same third doc. (L, undirected, 
  graded by count)
  - Common-neighbor / Jaccard вҖ” overlap of full neighborhoods, undirected. (L, 
  undirected, graded)
  - Geodesic proximity вҖ” short path between them in the graph. (L, undirected, 
  graded by distance)
  - Community co-membership вҖ” fall in the same densely-connected cluster. (L, 
  undirected, graded)
  - Hub/authority role вҖ” a doc many point to is central; relatedness to the 
  corpus, not to one peer. (L, directed-derived, graded) вҖ” a node property, but
  it conditions every edge's weight.

  5. Temporal вҖ” the timestamps

  - Creation proximity (arcs) вҖ” made close in time, sharing the intent then
  underway. (L, undirected, graded by О”t)
  - Modification co-occurrence вҖ” edited in the same window/session. (L, 
  undirected, graded)
  - Lifecycle-phase alignment вҖ” both in genesis vs build vs refactor of the same
  arc. (L, undirected, graded)
  - Sequence/ordering вҖ” explicit "next/prev" or implicit succession. (E or L, 
  directed)

  6. Provenance вҖ” version control and authorship

  The corpus is under git; this channel is sitting unused.
  - Change coupling (co-commit) вҖ” changed together in the same commit. This is
  logical coupling from software-repository mining and is often a stronger
  relatedness predictor than content. (L, undirected, graded by co-change 
  frequency)
  - Same author вҖ” (L, undirected, graded by author prolificacy)
  - Commit-message reference вҖ” a commit ties two docs in prose. (L/E, 
  undirected)
  - Branch co-membership вҖ” born or living on the same branch. (L, undirected)

  7. Spatial вҖ” the filesystem

  The thing your doc argues against as an ontology is still a weak relatedness
  signal.
  - Same directory / sibling вҖ” (L, undirected, binary)
  - Path-name token overlap вҖ” (L, undirected, graded)
  - Tree distance вҖ” (L, undirected, graded)

  8. Behavioral вҖ” usage, if you ever log it

  Not present today, but it's a real channel and worth naming for completeness.
  - Co-retrieval вҖ” surfaced together by the same query. (L, undirected, graded)
  - Co-access вҖ” read together in the same agent session (access coupling). (L, 
  undirected, graded)

  The cross-cutting axes

  Independent of channel, every signal varies along axes worth making explicit,
  because they determine how the graph may use it:

  - Authored vs emergent вҖ” your explicit/latent split. Authored = truth,
  ungraded, can't be "wrong," can't be discovered. Emergent = graded, rankable,
  falsifiable.
  - Directed vs undirected вҖ” directionality is itself information (reference vs
  co-citation); collapsing it loses bits.
  - Binary vs graded вҖ” only graded signals have a threshold knob and a
  recall/precision tradeoff.
  - Symmetric vs asymmetric вҖ” embedding distance is symmetric; "supersedes" is
  not; PMI is symmetric but conditional probability isn't.
  - Transitive vs not вҖ” similarity is roughly transitive (enables clustering);
  typed edges usually aren't.
  - Value-borne vs structure-borne вҖ” does the signal live in the
  content/metadata (В§1вҖ“3,5вҖ“7) or in the shape of the graph (В§4)? Your doc's
  sharpest line вҖ” "signal read off the links themselves, not the values" вҖ”
  generalizes: В§4 is the only family that needs no document content at all.

  The unifying statement

  There is one relatedness signal вҖ” mutual information вҖ” observed through many
  channels. The explicit channels (В§1вҖ“2) deliver it at full confidence and zero
  recall beyond what was typed. Every other channel trades confidence for reach,
  and each is a different projection of the documents into a feature space
  where co-occurrence of rare features betrays a dependency the author never
  declared. The differentiator your doc claims for the graph holds and can be
  stated precisely: grep/FTS/LLM-reading can only harvest channels В§1вҖ“3; 
  channels В§4 (topology), В§5 (time), В§6 (provenance), and В§8 (usage) exist only 
  once the corpus is a graph with history. That's the unique territory.
