# Link form

Links use `[text](path)` form. The link text names what the target is, not `click here` or `this page`.

## How to apply

- Good: `see [positive-transforms.md](path/to/positive-transforms.md) for transform examples`.
- Bad: `for transform examples, [click here](path/to/positive-transforms.md)`.
- Bad: `more info on [this page](path/to/page.md)`.

## Why

The link text is what the reader scans. Descriptive link text tells the reader what they'll get without clicking; "click here" tells them nothing. Accessibility tools also surface link text in isolation — a list of "click here" links is useless.
