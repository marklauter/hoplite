# Positive transforms

Lookup material for writing-prose's golden rule. Consult this catalog when a draft contains negation and you need a model for rewriting positively.

## How to use

1. Scan the draft for trigger words — `not`, `don't`, `didn't`, `wasn't`, `won't`, `can't`, `cannot`, `never`, `avoid`, `should not`, plus phrase patterns `didn't seem`, `wasn't very`.
2. For each hit, ask: "What is this sentence actually trying to say?"
3. Rewrite to state that directly. The examples below are models.

## Examples

### Cinderella (from armlinhouse.com)

- Weak: The thought of attending the royal ball didn't excite Cinderella.
- Strong: The thought of attending the royal ball filled Cinderella with dread.

The rewrite delivers emotion directly rather than describing its absence.

### Editorial maxim

- Weak: Don't bury the lede.
- Strong: Front-load the lede.

The instruction names what to do, not what to avoid.

### Code behavior

- Weak: This function doesn't return null.
- Strong: This function always returns a value.

The rewrite states the positive guarantee rather than the negative one.

### Stylistic instruction

- Weak: Avoid passive voice.
- Strong: Write in active voice.

The rule states the target style, not the antipattern.

## When to keep the negation

Define-by-presence's natural-negation exception applies when:

- The constraint is named by its negation in the trade (the well-established idiom).
- The positive label is more confusing than the natural negative.
- A list of examples uses "avoid" as a label for things to recognize.

Otherwise, rewrite positively.
