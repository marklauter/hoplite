# Cross-document cohesion read

Compare against sibling artifacts. Do register, terminology, and structural pattern hold across them?

## How to apply

- Identify the sibling set (other artifacts in the same directory, same wiki section, or same collection).
- Check register: does the prose match what siblings use?
- Check terminology: do the same concepts use the same words?
- Check structural pattern: does this artifact follow the canonical shape its siblings established?
- If any answer is "no," fix the outlier — usually this artifact; sometimes a sibling.

## When this matters most

- New artifact in an established collection (alignment with siblings is the point).
- After a terminology change anywhere in the project (pair with **Rename propagation**).
- When the artifact type is new and this instance sets the pattern (pair with **The first page sets the pattern**).
