# Post-1.0 hygiene — README, editorial slim, cat rules, docs recompose

README development layout adds `using-graph`; editorial-principles drops 49 reference files for the slim notes/journal use case; cat injection inside components gets forbidden so sed pipes drop from skills; `docs/mcp` recomposes into `docs/hoplite` as architecture + tool-api + roadmap with notes triage; venv-bootstrap race documented with the design decision behind the workaround.

## Intent

1.0.0 shipped. The repo has shape, but several inherited or transitional artifacts need cleanup:

- README development layout still talks about the old skill set, missing `using-graph`.
- The editorial-principles component was authored for the general writing-prose case. For notes and journal entries — short-form, plainspoken, one-cycle artifacts — many of the 50+ reference files in the component tree are noise.
- Components injecting other components via cat (the nested-injection pattern) had been working, but Mark hit a Claude Code permission-prompt issue when the inner cat used `${CLAUDE_PLUGIN_ROOT}`. The fix is to forbid cat injection inside components entirely; components are content fragments, not composition points.
- `docs/mcp/` carried the legacy spec — separate data-model, behavior, implementation, decision-log, plus historical pivot notes. With day one shipped, the spec deserves to be recomposed as a current-state architecture doc, not a layered history.
- The venv-bootstrap race needed an actual write-up — both the race shape and the design decision behind the current SessionStart-based workaround.

## What landed (chronological)

- 2026-05-25 20:42 — README: development layout lists `using-graph` alongside `taking-notes` and `journaling`.
- 2026-05-25 20:45 — Editorial-principles: slim for notes/journal — drop 49 reference files. The component tree pruned from 50+ reference files down to ~3. The reference files were exhaustive expansions of each editorial bullet; notes and journal entries don't need that depth on every load. The remaining references stay: positive transforms, the few that the journaling and taking-notes skills explicitly reference.

The compound rule: components contain no `!cat` lines and no `${CLAUDE_PLUGIN_ROOT}` references in their bodies. The host SKILL.md does all the cat injections; the host SKILL.md resolves all the paths. Components become pure markdown fragments. Skills can drop the `sed | cat` pipes that were the workaround for the permission-gate issue.

Both consumer skills now have a single H2 (`## Shape and editorial guidance`) under which all four cat-injections live: artifact-structure, frontmatter, mcp-reference, writing-prose. Visual alignment makes the composition pattern legible.

The race shape gets a proper write-up at `docs/notes/venv-bootstrap-races-mcp-supervisor-on-plugin-install.md`; the design decision behind the SessionStart workaround gets recorded so future readers don't have to reconstruct it.

## Decisions captured

- Components are content fragments. No cat injection inside components. No `${CLAUDE_PLUGIN_ROOT}` in component bodies. The host SKILL.md owns composition and path resolution.
- Editorial-principles is for notes and journal, not for the broader skill family. The slim was right for the consumers that load it. (If `writing-csharp` or `reviewing-prose` needed the fuller reference set, those skills would carry their own component or load a richer one.)
- Day-one architecture deserves one document, not three. The contract/implementation split from the spec drafting era served the design phase; once the design is settled, the split adds reading friction. `architecture.md` carries the full current state.
- `_archive/` is the history folder. Historical context with no day-one role moves under `_archive/`. The directory is gitignored so it can hold copies of files from other repos (like the docs/mcp copy from the Claude repo) without polluting git's view of the project.
- Recompose pass that touches stale doc paths must sweep everything. Source files, READMEs, CLAUDE.md, pyproject.toml docstrings, test files — any reference to `docs/mcp/<file>.md` had to update in the same change-set. The cascade is large; missing any one leaves a dangling link.
- Venv-bootstrap race is a known limitation with a documented workaround, not a pending fix. The fix needs upstream changes to Claude Code's SessionStart timing on `/plugin install`. Documenting the workaround (start a new session after install) lets users get unstuck without the project owning a fix it can't ship.

## What this closes

- The repo split started a few hours earlier. After this session, the hoplite repo is genuinely self-contained: docs, code, skills, tests, journal — everything lives under one tree, with a clear architecture doc, a clear roadmap, a clear separation between live notes and archive.
- The transition-era confusion. Several days of design pivot, multiple spec versions, three names for the package, two repos in flight at once. The recompose put a clear current-state surface on top so a reader (or a future agent) can find ground.
